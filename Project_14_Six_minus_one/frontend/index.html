CogniLens

Inspect cognitive load before users feel it.

Start from the page you are actively building. CogniLens checks readability, information overload, distraction, and structural consistency from a cognitive accessibility perspective.

Choose input workflow

How do you want to inspect the page?

URL analysis is the main developer workflow. Static upload stays available when you only have exported files.

Analyze a running page

Paste a local dev server or website URL, then re-run the analysis after each design change without exporting files. Client-rendered React/Vue pages may still need a rendered-browser snapshot in a later version.

Examples: localhost:5173 · localhost:3000 · 127.0.0.1:5500/index.html

Analyze a static package

Use HTML for a quick static check, or ZIP for an offline package containing HTML, CSS, and JavaScript.

Developer loop

Run, inspect, revise, repeat.

Designed for quick iteration: open your working page, analyze cognitive barriers, adjust the design, then refresh the result. The current MVP reads returned HTML plus linked CSS/JS signals.
