How to use CogniLens

CogniLens helps web designers and developers identify cognitive accessibility issues, inspect where they appear, and turn the findings into redesign actions.

1

Start a new analysis

Click New Analysis, then analyse a website URL, localhost page, HTML file, or ZIP package.

After analysis, CogniLens opens the dashboard with issue categories, issue cards, and an issue workspace.

2

Use risk levels to prioritise

High, Medium, and Low risk show which issue categories may need attention first.

Risk levels are a prioritisation guide, not a final accessibility grade. A low-risk category can still contain issues worth reviewing.

3

Work through one issue at a time

The issue cards on the left are the main workflow. Each card gives a short issue summary and two actions:

  • Show highlighted location opens the website preview and highlights where the issue appears.
  • Read guidance opens the right-side issue guidance with redesign steps, cognitive impact, evidence, affected users, and standards mapping.
4

Read guidance before redesigning

Use Recommended fix steps as the first redesign action. Then read Why this matters to understand the cognitive load impact.

Page evidence summarises key affected page areas. Technical CSS selectors are kept in optional developer details.

5

Reopen previous work

Use History to reopen previous analyses and compare later design changes with earlier versions.
