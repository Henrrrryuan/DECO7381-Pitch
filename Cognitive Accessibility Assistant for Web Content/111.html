Problem Space & Ethical Considerations

Cognitive Accessibility Assistant for Web Content

Web accessibility often focuses on sensory or motor impairments, while cognitive accessibility for neurodivergent users remains under-addressed. This project aims to identify cognitive load sources and provide clear, actionable recommendations.

Core Functions · Cognitive Accessibility Analysis

What we analyse across the page to detect cognitive load risks

Language Complexity

Checks whether key content is readable, concrete, and cognitively light enough for target users.

  • Score paragraphs by reading level, sentence length, and structure complexity.
  • Highlight jargon, idioms, and dense technical clusters that may overload working memory.
  • Prioritise instructions, forms, and error messages with stricter readability thresholds.

Standard / User Need Alignment

WCAG 3.1.5 Reading Level · ISO 9241‑11 (Effectiveness for comprehension) · Supports users with dyslexia & ADHD

Layout & Visual Hierarchy

Quantifies how layout, spacing, and hierarchy support quick scanning and orientation.

  • Measure layout density and whitespace ratio for different page regions.
  • Check heading hierarchy, grouping, and alignment for consistent visual entry points.
  • Detect cluttered zones where competing elements fragment attention.

Standard / User Need Alignment

WCAG 1.3.1 Info and Relationships · ISO 9241‑11 (Efficiency & safety) · Supports users with executive function challenges

Animation & Media

Assesses motion and media patterns that may distract or overwhelm neurodivergent users.

  • Detect auto‑play video/audio and looping animated elements.
  • Flag high‑frequency flashing and rapid motion that may trigger discomfort.
  • Identify non‑dismissible or constantly moving banners and overlays.

Standard / User Need Alignment

WCAG 2.2.2 Pause, Stop, Hide · WCAG 2.3.1 Seizures and Physical Reactions · Supports sensory‑sensitive users

Interaction Complexity

Evaluates how many steps, decisions, and state changes are required to complete tasks.

  • Count interaction steps and required fields for key conversion / task flows.
  • Detect unclear feedback such as hidden errors or missing confirmations.
  • Analyse navigation patterns for detours, dead ends, and inconsistent routes.

Standard / User Need Alignment

WCAG 3.2 Predictable · ISO 9241‑11 (Efficiency & satisfaction) · Supports users with limited working memory & cognitive fatigue

Insight & Recommendation Panel

Language Complexity・Detected Issues

Long, nested sentences and dense jargon were detected in key instructional blocks, increasing working memory demands.

Standard mapping: WCAG 3.1.5 Reading Level · ISO 9241‑11 (Effectiveness for comprehension)

  • Rewrite high‑impact sections (forms, error messages) to a lower reading level.
  • Replace cluster of domain jargon with plain language examples.
  • Add summaries at the top of complex pages for quick scanning.

Layout & Visual Hierarchy・Detected Issues

High layout density and inconsistent heading hierarchy in the hero area slow down orientation for first‑time visitors.

Standard mapping: WCAG 1.3.1 Info and Relationships · ISO 9241‑11 (Efficiency & safety)

  • Increase spacing around primary call‑to‑action and key headings.
  • Normalise heading levels (H1/H2/H3) to reflect logical structure.
  • Group related controls into visually separated blocks.

Animation & Media・Detected Issues

Auto‑playing hero videos and multiple simultaneous motion elements compete for attention and may distract neurodivergent users.

Standard mapping: WCAG 2.2.2 Pause, Stop, Hide · WCAG 2.3.1 Seizures and Physical Reactions

  • Disable auto‑play by default, or provide a visible pause control.
  • Limit concurrent motion to a single focal element per view.
  • Offer a “reduced motion” preference that simplifies transitions.

Interaction Complexity・Detected Issues

Key tasks require many steps and offer limited feedback, making recovery from errors harder for users with cognitive fatigue.

Standard mapping: WCAG 3.2 Predictable · ISO 9241‑11 (Efficiency & satisfaction)

  • Reduce redundant fields and optional steps in main task flows.
  • Provide inline error messages with clear next actions.
  • Ensure navigation remains consistent across equivalent pages.

Controlled Comparison

Run A/B tests on baseline vs assistant‑supported pages.

  • Randomly assign participants to two conditions.
  • Measure time‑on‑task, completion, and error rates.

NASA‑TLX & Subjective Load

Use NASA‑TLX after each task block to capture perceived workload.

  • Record mental demand, effort, frustration, and temporal demand.
  • Compare scores between Baseline and Assistant conditions.

Task Completion Metrics

Track how the assistant changes task success and efficiency.

  • Completion rate, time on task, and error rate per scenario.
  • Status tags (Improved / Stable / Needs work) per dimension.

Evaluation Report

Generate a concise evaluation report combining all metrics.

  • Summary of detected issues and applied recommendations.
  • Before/after comparison for each cognitive dimension.
  • Exportable report for supervisors or client stakeholders.
